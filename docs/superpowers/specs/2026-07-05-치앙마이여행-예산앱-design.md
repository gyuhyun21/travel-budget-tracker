# 치앙마이 여행 예산/지출 관리 앱 설계

## 목적
치앙마이 여행 중 예산 대비 실제 지출을 혼자 기록하고 확인하기 위한 개인용 웹앱.

## 사용 조건
- 사용자: 본인 1인만 사용 (로그인/계정 불필요)
- 기기: 휴대폰 브라우저 1대에서만 사용 (기기 간 동기화 불필요)
- 오프라인에서도 열람/입력 가능해야 함 (여행 중 데이터 연결 불안정 대비)

## 기술 스택
- 바닐라 HTML/CSS/JS 단일 페이지 앱 (빌드 도구, 프레임워크 없음)
- PWA(Progressive Web App)로 구성 — `manifest.json` + 서비스워커로 홈 화면 설치 및 오프라인 캐싱 지원
- 데이터 저장은 브라우저 `localStorage` 사용 (별도 서버/DB 없음)

## 데이터 모델

### Settings (localStorage에 단일 객체로 저장)
| 필드 | 타입 | 설명 |
|---|---|---|
| totalBudget | number | 총 예산 (원화 기준) |
| thbRate | number | 1바트(THB) = ?원 고정 환율 |
| usdRate | number | 1달러(USD) = ?원 고정 환율 |
| tripStartDate | string (YYYY-MM-DD) | 여행 시작일 |
| tripEndDate | string (YYYY-MM-DD) | 여행 종료일 |

### Expense (localStorage에 배열로 저장)
| 필드 | 타입 | 설명 |
|---|---|---|
| id | string | 고유 식별자 (생성 시 timestamp 기반) |
| date | string (YYYY-MM-DD) | 지출 날짜 |
| amount | number | 입력한 원본 금액 |
| currency | "THB" \| "USD" \| "KRW" | 입력 통화 |
| krwAmount | number | 환율 적용하여 계산된 원화 환산 금액 (저장 시점 환율 기준으로 계산, 이후 환율 변경해도 과거 기록은 불변) |
| memo | string | 메모 (선택 입력) |

## 화면 구성

### 1. 대시보드 (메인 화면)
- 남은 예산 (총예산 - 총지출)
- 총 지출액
- 예산 대비 지출 진행률 바 (%)
- 여행 진행률 (예: "여행 3일차 / 총 5일" 또는 D-day 표시, 시작일 이전이면 D-N으로 표시)
- 최근 지출 내역 리스트 (최신순, 날짜/금액/통화/원화환산/메모 표시)
- "지출 추가" 버튼 → 지출 추가 화면으로 이동

### 2. 지출 추가/수정 화면
- 날짜 입력 (기본값: 오늘)
- 통화 선택 (THB/USD/KRW 중 택1)
- 금액 입력
- 메모 입력 (선택)
- 저장 시 설정된 환율로 원화 환산값(krwAmount) 계산하여 저장
- 수정 화면은 기존 값 프리필, 삭제 버튼 포함

### 3. 지출 목록 화면
- 전체 지출 내역을 날짜순으로 조회
- 각 항목 클릭 시 수정 화면으로 이동, 삭제 가능

### 4. 설정 화면
- 총 예산 입력
- THB→KRW 환율 입력
- USD→KRW 환율 입력
- 여행 시작일/종료일 입력
- 데이터 백업: JSON 파일로 내보내기 버튼
- 데이터 복원: JSON 파일 불러오기 버튼 (기존 데이터 덮어쓰기 전 확인)

## 에러 처리 / 예외 상황
- 환율이 설정되지 않은 상태에서 THB/USD 지출 입력 시: krwAmount는 0으로 계산하지 않고, 설정 화면으로 유도하는 안내 표시
- JSON 불러오기 시 형식이 올바르지 않으면 에러 메시지 표시하고 기존 데이터 유지
- localStorage 데이터가 없는 최초 실행 시: 설정 화면으로 자동 유도하여 예산/환율 입력 요청

## 검증 계획
자동화 테스트 프레임워크는 사용하지 않음 (개인용 단일 페이지 앱 규모에 비해 과함). 대신 브라우저에서 다음 흐름을 수동으로 확인:
1. 최초 실행 시 설정 화면 유도 확인
2. 예산/환율/여행일자 설정 후 대시보드에 반영되는지 확인
3. THB/USD/KRW 각 통화로 지출 추가 시 원화 환산이 올바른지 확인
4. 지출 수정/삭제가 목록과 대시보드 합계에 정상 반영되는지 확인
5. 여행 진행률(D-day/N일차) 표시가 날짜에 따라 올바르게 계산되는지 확인
6. JSON 내보내기 후 불러오기로 데이터가 정상 복원되는지 확인
7. 홈 화면에 PWA로 설치되고 오프라인에서도 열리는지 확인

## 범위 밖 (Out of Scope)
- 다중 사용자/기기 간 동기화
- 실시간 환율 API 연동
- 카테고리별 예산 구분 (지출 자체에도 카테고리 태그 없음, 메모로만 구분)
