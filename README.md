# 여행 가계부

혼자 쓰는 예산/지출 관리 웹앱입니다. 기본은 서버나 로그인 없이 브라우저 `localStorage`에 데이터를 저장하고, 설정 화면에서 원하는 제목을 입력하면 여행, 행사 등 어떤 용도로든 이 앱 하나로 관리할 수 있습니다. 설정 화면에서 "공유하기"를 누르면 링크를 아는 사람끼리 실시간으로 같이 보고 편집할 수도 있습니다.

## 실행 방법

```bash
cd "이 폴더 경로"
python3 -m http.server 8080
```

브라우저에서 `http://localhost:8080/index.html` 접속. 모바일 크롬/사파리에서는 "홈 화면에 추가"로 설치해서 앱처럼 쓸 수 있습니다.

## 데이터 백업

설정 화면에서 "JSON으로 내보내기"로 백업 파일을 받아두세요. 브라우저 캐시를 지우면 데이터가 사라질 수 있습니다.

## 영수증 촬영/OCR 자동 등록

지출 추가 화면의 "영수증으로 등록" 버튼으로 영수증 사진을 찍거나 업로드하면, 금액/가게이름/카테고리를 자동으로 인식해 입력해줍니다. 인식 결과는 저장 전에 항상 검토·수정할 수 있습니다.

- OCR 엔진(Tesseract.js)은 CDN에서 불러오므로 **첫 사용 시 인터넷 연결이 필요**합니다. 예산/지출 관리 등 나머지 핵심 기능은 완전히 오프라인으로 동작합니다.
- 인식 정확도는 영수증 화질/각도에 따라 달라질 수 있어, 자동 채워진 금액과 카테고리는 저장 전에 확인하는 걸 권장합니다.

## 실시간 공유 (같이 편집)

설정 화면 → "공유" 섹션 → "이 여행 공유하기"를 누르면 Firebase(Firestore)에 지금까지의 예산/지출 데이터가 올라가고, 접속 주소 끝에 `?trip=...`이 붙은 공유 링크가 생성됩니다. 이 링크를 받은 사람은 누구나 같은 여행 데이터를 실시간으로 같이 보고 편집할 수 있습니다.

- **접근 방식**: 로그인/계정 없이 "링크를 아는 사람은 누구나 편집 가능"한 구조입니다 (구글 문서의 "링크가 있는 모든 사용자" 공유와 동일한 방식). 링크가 유출되면 그 사람도 데이터를 보고 고칠 수 있으니, 신뢰하는 사람에게만 링크를 공유하세요.
- 공유를 시작하면 인터넷 연결이 필요합니다. "공유 중지" 버튼으로 언제든 그 기기만 로컬 저장으로 되돌릴 수 있고, 이후에도 다른 참여자들은 계속 실시간 공유를 이어갈 수 있습니다.
- Firestore 콘솔의 보안 규칙(테스트 모드는 30일 후 자동 만료됨)은 아래로 교체해서 계속 쓸 수 있게 해두세요:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /trips/{tripId} {
      allow read, write: if true;
      match /expenses/{expenseId} {
        allow read, write: if true;
      }
    }
  }
}
```
