# 여행 가계부

혼자 쓰는 예산/지출 관리 웹앱입니다. 기본은 서버나 로그인 없이 브라우저 `localStorage`에 데이터를 저장하고, 설정 화면에서 원하는 제목을 입력하면 여행, 행사 등 어떤 용도로든 이 앱 하나로 관리할 수 있습니다. 설정 화면에서 "공유하기"를 누르면 링크를 아는 사람끼리 실시간으로 같이 보고 편집할 수도 있습니다.

## 실행 방법

```bash
cd "이 폴더 경로"
python3 -m http.server 8080
```

브라우저에서 `http://localhost:8080/index.html` 접속. 모바일 크롬/사파리에서는 "홈 화면에 추가"로 설치해서 앱처럼 쓸 수 있습니다.

## 데이터 백업

설정 화면에서 "JSON으로 내보내기"로 백업 파일을 받아두세요. 브라우저 캐시를 지우면 데이터가 사라질 수 있습니다.

## 영수증 촬영/OCR 자동 등록

지출 추가 화면의 "영수증으로 등록" 버튼으로 영수증 사진을 찍거나 업로드하면, 금액/가게이름/카테고리를 자동으로 인식해 입력해줍니다. 인식 결과는 저장 전에 항상 검토·수정할 수 있습니다.

- OCR 엔진(Tesseract.js)은 CDN에서 불러오므로 **첫 사용 시 인터넷 연결이 필요**합니다. 예산/지출 관리 등 나머지 핵심 기능은 완전히 오프라인으로 동작합니다.
- 인식 정확도는 영수증 화질/각도에 따라 달라질 수 있어, 자동 채워진 금액과 카테고리는 저장 전에 확인하는 걸 권장합니다.

## 지출한 사람별 합계

지출 추가/수정 화면에서 "지출한 사람"을 참여자 중에서 선택할 수 있습니다 (참여자 목록은 "준비물" 탭의 "참여자 관리"에서 등록한 것과 같습니다). 아무도 선택하지 않으면 "미확인"으로 기록됩니다.

- "지출 목록" 화면 상단에 참여자별 총 지출 금액이 카드로 표시됩니다. 참여자는 노란색 그라데이션, "미확인"은 준비물 탭의 "미정" 칸과 같은 핫핑크 그라데이션으로 구분됩니다.
- 이미 등록한 지출도 다시 열어서 지출한 사람을 바꾸거나 지정할 수 있습니다.

## 실시간 공유 (같이 편집)

설정 화면 → "공유" 섹션 → "이 여행 공유하기"를 누르면 Firebase(Firestore)에 지금까지의 예산/지출 데이터가 올라가고, 접속 주소 끝에 `?trip=...`이 붙은 공유 링크가 생성됩니다. 이 링크를 받은 사람은 누구나 같은 여행 데이터를 실시간으로 같이 보고 편집할 수 있습니다.

- **접근 방식**: 로그인/계정 없이 "링크를 아는 사람은 누구나 편집 가능"한 구조입니다 (구글 문서의 "링크가 있는 모든 사용자" 공유와 동일한 방식). 링크가 유출되면 그 사람도 데이터를 보고 고칠 수 있으니, 신뢰하는 사람에게만 링크를 공유하세요.
- 공유를 시작하면 인터넷 연결이 필요합니다. "공유 중지" 버튼으로 언제든 그 기기만 로컬 저장으로 되돌릴 수 있고, 이후에도 다른 참여자들은 계속 실시간 공유를 이어갈 수 있습니다.
- Firestore 콘솔의 보안 규칙(테스트 모드는 30일 후 자동 만료됨)은 아래로 교체해서 계속 쓸 수 있게 해두세요:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /trips/{tripId} {
      allow read, write: if true;
      match /expenses/{expenseId} {
        allow read, write: if true;
      }
      match /meals/{mealId} {
        allow read, write: if true;
      }
      match /packingItems/{itemId} {
        allow read, write: if true;
      }
    }
  }
}
```

## 뭐먹? (일정별 식사 추천)

하단 탭의 "뭐먹?"에서 여행 일자별로 조식/중식/석식/간식을 나눠서, 같이 여행하는 사람들이 각자 추천하는 식당이나 메뉴를 추가/삭제할 수 있습니다. 공유 중인 여행이면 다른 사람이 추가한 추천도 실시간으로 바로 보입니다.

- 추가 시 "식당" 모드에서는 카카오맵 검색으로 식당을 등록하면 주소가 같이 저장되고, 항목의 "지도" 버튼으로 카카오맵 페이지가 새 창에서 열려 위치와 상세 정보를 확인할 수 있습니다.
- 캠핑처럼 갈 식당이 없을 때는 "메뉴만" 모드로 전환하면 검색/주소 없이 메뉴·음식 이름만 입력할 수 있습니다.
- 카카오맵 연동에는 카카오 디벨로퍼스의 **JavaScript 키**가 필요합니다 (`js/kakao-map.js`의 `KAKAO_JS_KEY`에 설정). 키가 없어도 식당/메뉴 이름을 직접 입력하는 방식으로 계속 사용할 수 있습니다.
- 카카오 디벨로퍼스 앱의 플랫폼 설정(Web)에 이 사이트 도메인(예: `gyuhyun21.github.io`, `localhost:8080`)을 등록해야 검색이 동작합니다.

## 준비물

하단 탭의 "준비물"에서 여행에 필요한 물건을 등록하고 담당자를 정할 수 있습니다.

- "참여자 관리"에서 준비물을 나눠 가져올 사람 이름(예: 규현, 인혁, 태균)을 등록하면 화면 상단에 표시됩니다.
- 준비물을 추가하면 표에 새 행으로 나타나고, 참여자(열)와 준비물(행)이 만나는 칸을 탭해서 담당자를 지정합니다. 이미 담당이 정해진 칸을 다시 탭하면 준비 완료로 체크됩니다. 준비물 이름을 탭하면 이름/메모 수정과 삭제가 가능합니다.
- 각 항목의 동그라미를 누르면 준비 완료로 체크되고, 상단에 "N / 전체 준비완료" 진행 상황이 표시됩니다.
- 공유 중인 여행이면 참여자, 준비물, 체크 상태 모두 실시간으로 같이 보고 편집됩니다.
